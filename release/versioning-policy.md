# Versioning Policy

The Sentient OS Design Genome uses semantic versioning.

## MAJOR

Breaking changes to:

- import order
- public exports
- data attributes
- schema requirements
- component contracts
- token names

## MINOR

Additive changes such as:

- new approved components
- new personalities
- new architectural families
- new project DNA entries
- additive tokens

## PATCH

Non-breaking changes such as:

- documentation fixes
- validator improvements
- non-breaking CSS fixes
- preview updates

## Breaking Changes

The following are breaking changes:

- renaming CSS variables
- renaming primitive classes
- removing public exports
- changing required data attributes
- changing import order
- removing approved component contracts
- changing schema-required fields

## Governance Rule

No breaking change may be introduced without changelog entry and migration notes.
