# Import Order Contract

The stable v1.0 import order is:

```html
<link rel="stylesheet" href="dist/sentient-foundation.css">
<link rel="stylesheet" href="dist/sentient-primitives.css">
<link rel="stylesheet" href="dist/sentient-personalities.css">
<link rel="stylesheet" href="dist/sentient-architectures.css">
<link rel="stylesheet" href="dist/sentient-projects.css">
```

Layer meanings:

- Foundation = immutable tokens and base variables
- Primitives = neutral component structure
- Personalities = atmosphere and mood
- Architectures = spatial behavior
- Projects = Project DNA composition and chapter accent bridge

Changing import order is a breaking change.

Future application CSS must come after the Design Genome imports.

Application CSS may extend consumption-specific behavior, but it may not:

- override Foundation variables
- rename primitive classes
- bypass component contracts
- erase contract-owned primitive behavior
