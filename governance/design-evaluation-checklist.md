# Design Evaluation Checklist

Use this checklist before approving any new screen, theme, or major refactor.

## Architecture

- Does it improve orientation?
- Does it improve hierarchy?
- Does it improve circulation?
- Does it improve understanding?
- Does it improve focus?
- Does it communicate one clear primary workspace?
- Does it feel constructed rather than assembled from unrelated cards?

## System Fit

- Does it strengthen the Sentient OS ecosystem?
- Does it fit the declared architectural family?
- Does it respect the selected personality?
- Does it preserve Foundation?
- Does it avoid introducing a new isolated visual language?
- Does it preserve predictable navigation?

## Theme Governance

- Does it support synced theme declarations?
- Does it keep spacing and typography hierarchy intact?
- Does it preserve component anatomy and status semantics?
- Does it avoid changing keyboard and responsive behavior?

## Accessibility

- Does it meet WCAG AA contrast minimums?
- Are focus states always visible?
- Is keyboard navigation supported?
- Is reduced motion supported?
- Are states immediately legible?

## Decision Rule

If any critical answer is no:

- remove it
- simplify it
- refactor it
- or document it as a local exception
